Data Flow

  1. Source Capture: Three connectors capture customer changes in real-time (CDC) or near real-time (polling)
  2. Topic Ingestion: Raw events published to separate Kafka topics with Avro schemas
  3. Stream Processing: PyFlink merges streams, detects changes, implements Type 6 SCD logic
  4. PostgreSQL Sink: Processed data written to three analytical tables optimized for different query patterns

Target Latency: < 2 minutes end-to-end | POC Duration: 30 days | Entity: Customer master data only
