# Update Documentation

Update documentation based on recent changes or new information.

## Steps

1. **Identify what needs updating**
   - New feature or process?
   - Changed existing functionality?
   - Correction to existing docs?

2. **Locate relevant documents**
   - Check `index.yaml` for existing docs
   - Find related domain/product folders
   - Identify cross-references

3. **Make updates**
   - Update content following templates in CONTRIBUTING.md
   - Maintain consistent formatting
   - Add/update diagrams if needed

4. **Update cross-references**
   - Update links in related documents
   - Update README files in affected folders
   - Update `index.yaml` if new documents created

5. **Validate**
   - Run validation scripts if available
   - Check all links work
   - Verify formatting is correct

## Output
- Updated documentation files
- Updated index and cross-references
- Summary of changes made

## Notes
- Always check existing docs before creating new ones
- Follow the domain-driven structure
- Update glossary if new terms introduced
